## 测试步骤

### 1. 打开应用

应用窗口应该已经自动打开。如果没有：
- 在任务栏找到"项目管理助手"窗口
- 或者在浏览器打开 http://localhost:1420

### 2. 打开开发者工具

按 `F12` 或 `Ctrl+Shift+I` 打开开发者工具

切换到 **Console** 标签页，查看日志

### 3. 测试基础功能

#### 测试1：项目列表
- ✅ 应该看到项目列表
- ✅ 项目阶段标签应该显示新阶段（售前、启动、需求等）

#### 测试2：创建项目
- 点击"新建项目"按钮
- 输入项目名称
- 点击"创建"
- ✅ 应该看到成功提示
- ✅ 项目列表应该刷新

#### 测试3：打开项目
- 点击项目旁边的"打开"按钮
- ✅ 应该进入项目首页
- ✅ 左侧应该显示阶段导航
- ✅ 上方应该显示摘要卡片
- ✅ 下方应该显示文件拖拽区域

### 4. 测试文件上传

#### 测试1：拖拽上传
- 在项目首页，拖拽一个文件到上传区域
- ✅ 应该看到上传成功提示
- ✅ 控制台应该显示日志

#### 测试2：检查文件
- 检查文件是否保存到正确位置
- 路径：`C:\Users\kingdee\AppData\Roaming\com.kingdee.project-manager\projects\[项目名]\[阶段]\`

### 5. 测试AI对话

#### 测试1：进入对话
- 在项目首页，点击"AI对话"摘要卡片
- ✅ 应该进入对话页面

#### 测试2：发送消息
- 输入消息，点击"发送"
- ✅ 应该看到消息显示
- ✅ 应该看到AI回复（或错误提示）
- ✅ 控制台应该显示日志

### 6. 测试设置功能

#### 测试1：进入设置
- 点击右上角设置按钮（蓝色背景）
- ✅ 应该进入设置页面

#### 测试2：查看配置
- ✅ 应该看到AI配置项
- ✅ 应该看到文件存储位置信息

## 控制台日志说明

在控制台中，你会看到带 `[` 标签的日志：

- `[App]` - 页面渲染和导航
- `[FileDropZone]` - 文件拖拽上传
- `[fileService]` - 文件上传服务
- `[ChatWindow]` - AI对话
- `[aiService]` - AI服务

**特别关注：**
- ✅ 成功日志：绿色或白色
- ❌ 错误日志：红色

## 常见问题

### Q1: 窗口没有打开
A: 检查任务栏，或者在浏览器打开 http://localhost:1420

### Q2: 文件上传失败
A: 查看控制台日志，检查：
- 文件读取是否成功
- API 调用是否成功
- 错误信息是什么

### Q3: AI对话失败
A: 查看控制台日志，检查：
- API Key 是否配置正确
- 网络请求是否成功
- 错误信息是什么

### Q4: 设置按钮看不到
A: 设置按钮现在是蓝色背景白色文字，在右上角
