# 文件内容提取和AI分析 - 可行性分析

**日期：** 2026-06-05
**目标：** 评估使用云端大模型实现文件内容提取和AI分析的可行性

---

## 一、需求回顾

### 1.1 核心需求

1. **文件内容提取**：从各种格式的文件中提取文本内容
2. **AI智能分类**：根据文件内容（不是标题）进行分类
3. **AI生成MD文件**：分析项目文件，生成包含以下内容的MD文件：
   - 文件清单（位置、类型、大小）
   - 项目进展
   - 问题清单
   - 内容摘要
4. **AI对话上下文**：将MD文件作为上下文输入给AI

### 1.2 支持的文件类型

- Office文档（Word、PDF、Excel、PPT）
- 纯文本文件（Markdown、TXT、CSV）
- 图片文件（PNG、JPG）
- 代码文件

### 1.3 更新频率

- 手动/定时更新（非实时）

## 二、技术方案分析

### 2.1 文件内容提取方案

#### 方案A：本地提取

| 文件类型 | 本地库 | 可行性 |
|----------|--------|--------|
| TXT/MD/CSV | 直接读取 | ✅ 完全可行 |
| 代码文件 | 直接读取 | ✅ 完全可行 |
| PDF | pdf-parse / pdfjs-dist | ✅ 可行 |
| Word | mammoth / docx | ✅ 可行 |
| Excel | xlsx / exceljs | ✅ 可行 |
| PPT | pptx-parser | ⚠️ 部分可行 |
| 图片 | 需要OCR | ❌ 需要云端 |

**结论：** 除了图片，其他文件都可以本地提取文本内容。

#### 方案B：云端提取

- 使用云端OCR服务（如百度OCR、腾讯OCR）
- 支持所有格式，包括图片
- 需要网络和API费用

#### 方案C：混合方案（推荐）

- 文本文件：本地提取
- 图片文件：调用多模态大模型（如GLM-4.1V-Thinking-Flash）

## 三、可行性评估

### 3.1 文件内容提取

| 文件类型 | 提取方式 | 可行性 | 风险 |
|----------|----------|--------|------|
| TXT/MD/CSV | 本地读取 | ✅ 完全可行 | 无 |
| 代码文件 | 本地读取 | ✅ 完全可行 | 无 |
| PDF | 本地库 | ✅ 可行 | 复杂PDF可能提取不完整 |
| Word | 本地库 | ✅ 可行 | 格式可能丢失 |
| Excel | 本地库 | ✅ 可行 | 需要结构化处理 |
| PPT | 本地库 | ⚠️ 部分可行 | 提取可能不完整 |
| 图片 | 多模态模型 | ⚠️ 需要验证 | 免费API限制 |

### 3.2 AI智能分类

**实现方式：**
1. 提取文件文本内容
2. 调用GLM-4.7-Flash分析内容
3. 返回分类结果

**可行性：** ✅ 可行

**风险：**
- 免费API调用限制
- 大文件可能超出Token限制
- 分类准确率需要测试

### 3.3 AI生成MD文件

**实现方式：**
1. 收集项目所有文件的内容摘要
2. 调用GLM-4.7-Flash分析
3. 生成结构化的MD文件

**可行性：** ⚠️ 有风险

**风险：**
- 文件数量多时，可能超出Token限制
- 需要多次API调用，耗时较长
- 免费API调用次数可能不够

**解决方案：**
- 分批处理文件
- 使用本地摘要提取，减少Token消耗
- 限制单次处理的文件数量

### 3.4 AI对话上下文

**实现方式：**
1. 读取项目MD文件内容
2. 作为上下文发送给AI
3. AI基于上下文回答问题

**可行性：** ✅ 可行

**风险：**
- MD文件过大可能超出Token限制
- 需要截断或摘要处理

## 四、风险评估

### 4.1 高风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 免费API调用限制 | 功能无法使用 | 需要查证最新限制，准备付费方案 |
| 大文件超出Token限制 | 处理失败 | 分块处理、摘要提取 |
| 图片OCR效果差 | 分类不准确 | 使用多模态模型、限制图片类型 |

### 4.2 中风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 文件格式兼容性 | 部分文件无法处理 | 降级处理、提示用户 |
| API响应慢 | 用户体验差 | 异步处理、进度提示 |
| 生成的MD质量差 | 功能无用 | 优化Prompt、人工审核 |

### 4.3 低风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 网络不稳定 | 调用失败 | 重试机制、离线降级 |
| API Key泄露 | 安全问题 | 本地存储、不上传 |

## 五、建议方案

### 5.1 推荐方案：混合架构

```
文件上传 → 本地提取文本 → AI分析（云端） → 生成MD → 存储本地
```

**具体流程：**

1. **文件上传**
   - 保存到本地项目文件夹

2. **文本提取（本地）**
   - TXT/MD/CSV/代码：直接读取
   - PDF/Word/Excel：使用本地库提取
   - 图片：跳过或使用多模态模型

3. **AI分析（云端）**
   - 调用GLM-4.7-Flash
   - 生成分类、摘要、问题清单

4. **生成MD文件**
   - 保存到项目文件夹
   - 作为AI对话的上下文

### 5.2 降级方案

如果免费API限制太严格：

1. **只支持文本文件**
   - 不支持图片、复杂Office文档
   - 用户手动复制内容

2. **简化AI功能**
   - 只做简单分类
   - 不生成详细MD文件

3. **使用付费API**
   - 按量付费
   - 成本可控
